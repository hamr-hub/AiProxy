# 架构设计

## 分层架构

```
HTTP Request
    │
    ▼
┌─────────────────────────────────────────────────────────────┐
│                    Request Handler                          │
│               src/handlers/request-handler.js               │
├─────────────────────────────────────────────────────────────┤
│  1. CORS 处理                                                │
│  2. 静态文件服务                                              │
│  3. 插件路由分发                                              │
│  4. UI API 请求处理                                           │
│  5. 健康检查                                                  │
│  6. 提供商路径重写                                            │
│  7. 认证流程                                                  │
│  8. 中间件流程                                                │
│  9. API 请求分发                                              │
└─────────────────────────────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────────────────────────────┐
│                   Adapter Registry                         │
│                  src/providers/adapter.js                   │
├─────────────────────────────────────────────────────────────┤
│  ApiServiceAdapter 接口                                      │
│  ├── generateContent(model, requestBody)                    │
│  ├── generateContentStream(model, requestBody)              │
│  ├── listModels()                                          │
│  └── refreshToken()                                        │
└─────────────────────────────────────────────────────────────┘
```

## 核心模块

### 1. 主进程 (Master)

- 子进程管理（启动/停止/重启）
- 资源监控（CPU/内存阈值告警）
- 管理端口 3100

### 2. 提供商池管理器

- 健康状态管理
- LRU 评分选择策略
- 并发控制
- 故障转移链

### 3. 插件系统

- 生命周期管理
- 认证插件 / 中间件插件
- 路由扩展

---

## 设计模式

| 模式 | 应用 |
|------|------|
| 策略模式 | 提供商实现、转换器策略 |
| 适配器模式 | API 适配器封装 |
| 工厂模式 | 适配器实例创建 |
| 单例模式 | PluginManager |
| 观察者模式 | 事件广播 |
| 责任链模式 | 故障转移链 |
